## Data Integrity Notes

**CSV row count vs reported counts:** The `refined_all.csv` file contains 10,918 data rows, while this log reports 10,904 records after filtering. The `combined_deduplicated.csv` file contains 40,006 data rows vs 39,986 reported here. These discrepancies (14 and 20 rows respectively) arise because the CSV files and this log were generated by different runs of the pipeline. The screening pipeline operated on 10,904 records (confirmed by `screening_progress.yaml` and `SCREENING_COMPLETE.md`). The 14 extra records in the current CSV were not screened and do not affect downstream results.

**Batch CSV artifacts:** The 73 batch reconciled CSVs in `05_screening/` contain 11,084 total data rows (vs 10,904 expected). This excess is due to 289 rows with empty record IDs and 83 duplicate record IDs across batches — artifacts of the dual-screening reconciliation process. Arithmetic check: 11,084 total rows − 289 empty-ID rows = 10,795 non-empty rows = 10,712 unique IDs + 83 duplicate-instance rows. These artifacts do not affect the screening results, which are tracked by the definitive counts in `SCREENING_COMPLETE.md`.
